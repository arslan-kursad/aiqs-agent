# Phase-2B GPU-host detector stack (anomalib 2.x) — a SEPARATE environment.
#
# This is NOT a `uv` optional-extra. A co-locked extra in pyproject.toml is INFEASIBLE:
# the pinned local base caps are mutually exclusive with anomalib 2.x. Measured:
#
#   $ printf 'torchmetrics<1.5\nanomalib>=2.2\n' | uv pip compile -
#   x No solution found ... anomalib>=2.2.0 depends on torchmetrics>=1.8.2,
#     and because you require torchmetrics<1.5 ... requirements are unsatisfiable.
#
# (Same story for numpy<2 and lightning<2.5.) The base exists to keep the anomalib-1.2
# stack installable on the Intel-mac (no x86_64-macOS torch wheel past 2.2.2); anomalib 2.x
# only ever runs on a CUDA/arm64 GPU host. So install THIS on the GPU host, in a fresh env:
#
#   python -m venv .venv-ad2 && . .venv-ad2/bin/activate
#   pip install torch torchvision        # the CUDA build matching the host
#   pip install -r requirements-ad2.txt
#   pip install -e . --no-deps           # add the `aiqs` package WITHOUT the pinned base
#
# `--no-deps` is load-bearing: it stops pip from re-pulling the base's torch==2.2.2 / numpy<2
# pins. The detector code (detector.py/data.py) dispatches to `_detector_v2`/`_data_v2`
# automatically once anomalib 2.x is the installed version (see aiqs/_anomalib_compat.py).
#
# CLEANER LONG-TERM FIX (deferred — a base refactor, propose→confirm): make the detector
# stack itself an extra (base = the detector-free numpy/sklearn decision+VLM layer; `det1`
# and `ad2` as conflicting extras via `[tool.uv] conflicts`). That matches the thesis (the
# value layer needs no torch) and would let `uv sync --extra ad2` work natively.
